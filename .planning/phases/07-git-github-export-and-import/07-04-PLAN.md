| imported source metadata -> Data Sources and session routes | Imported archives must stay visibly distinct from live local roots and must never silently regain mutable operations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04-01 | Spoofing | imported-source identity | mitigate | Persist explicit `sourceKind`, `addedBy`, and archive metadata so imported sources cannot masquerade as normal local roots. |
| T-07-04-02 | Tampering | read-only source semantics | mitigate | Disable validate, scan, watch, git, and GitHub actions for imported archives in registry, DTO, and renderer layers, and assert that behavior in tests. |
| T-07-04-03 | Information Disclosure | archive-root access | mitigate | Allow archive-backed reads only through validated archive metadata and the shared safe-filesystem boundary; do not rebind imports to arbitrary host paths. |
| T-07-04-SC | Tampering | Phase 7 scope boundary | mitigate | Keep imports read-only and persistent; rebinding archives to live roots or making them writable stays out of scope. |
</threat_model>
