<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| validated git snapshot -> GitHub provider | GitHub lookups must not detach from validated repo context or trust unverified remotes. |
| shared `gh` commands -> route DTOs | Missing auth, no-match PR, and API failures must stay explicit and non-destructive to the broader triage experience. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Spoofing | GitHub repo and PR resolution | mitigate | Require validated git snapshot plus remote URL before any `gh` call and keep results project-scoped in shared cache. |
| T-07-02-02 | Denial of Service | `gh` process execution | mitigate | Use fixed read-only commands with conservative timeouts and degrade to Unknown or Unsupported states instead of blocking triage loads. |
| T-07-02-03 | Tampering | GitHub truth-state rendering | mitigate | Keep No Matching PR, Unknown, and Unsupported as first-class route states and forbid renderer-side polling or hidden fallback sections. |
| T-07-02-SC | Tampering | Phase 7 scope boundary | mitigate | Limit this plan to summary-only GitHub state; comments, timelines, and write actions stay out of scope. |
</threat_model>
