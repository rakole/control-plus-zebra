## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Gemini artifact files -> parser | Chat logs, sidecars, and metadata files are untrusted and may be partial, duplicated, malformed, or inconsistent. |
| parser -> shared core | Raw Gemini semantics must not leak into shared contracts or be misrepresented as normalized truth too early. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | chat/log parsing | mitigate | Parse line-by-line / record-by-record, emit diagnostics for malformed rows, and continue over remaining valid evidence. |
| T-04-02-02 | Repudiation | contradictory metadata | mitigate | Preserve auxiliary `logs.json` and `$set` patch evidence as raw events so later normalization can diagnose contradictions instead of hiding them. |
| T-04-02-03 | Denial of Service | sidecar parsing | mitigate | Treat unreadable, malformed, or unknown sidecars as bounded diagnostics rather than fatal parser errors. |
| T-04-02-SC | Tampering | npm/pip/cargo installs | mitigate | No package install is planned. If implementation attempts any install, stop and add a blocking human verification checkpoint because slopcheck was unavailable. |
</threat_model>
