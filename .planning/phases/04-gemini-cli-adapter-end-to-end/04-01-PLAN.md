<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| configured source root -> adapter discovery | User-configured temp-root paths are untrusted and may be missing, malformed, or point outside Gemini layouts. |
| adapter discovery -> filesystem | Discovery must not escape configured root boundaries or treat unrelated files as trusted Gemini evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | root validation | mitigate | Validate roots through `SafeFilesystem`, reject non-directories or missing paths with diagnostics, and never coerce missing roots into empty success. |
| T-04-01-02 | Information Disclosure | artifact discovery | mitigate | Enumerate only configured-root descendants and known Gemini artifact families; ignore unrelated noise such as `.DS_Store`. |
| T-04-01-03 | Spoofing | source and artifact identity | mitigate | Generate source/artifact IDs from adapter ID plus native project/artifact identity to avoid cross-project or cross-harness collisions. |
| T-04-01-SC | Tampering | npm/pip/cargo installs | mitigate | No package install is planned. If implementation attempts any install, stop and add a blocking human verification checkpoint because slopcheck was unavailable. |
</threat_model>
