# Phase 02 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase UI Scope

Phase 2 must render a quiet, utilitarian local desktop workbench shell. The first real route is Sessions-first: a list/detail preview of sanitized fake-adapter-backed session summaries loaded through the typed preload/IPC bridge.

The app may show persistent navigation placeholders for Overview, Projects, Sessions, and Diagnostics, but only Sessions is implemented in this phase. Do not add launch, approve/reject, terminal control, PR creation, cleanup, source mutation, or arbitrary shell execution flows.

Renderer copy and UI state must stay harness-neutral. Unsupported or unknown capabilities must render as explicit `Unsupported` or `Unknown` labels, never as `0`, `Passed`, `Clean`, or hidden values.

Primary visual anchor: the selected session row and its preview panel. Route title, placeholder navigation, and `Reload Sessions` control are secondary.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge icon spacing, table cell micro-gaps |
| sm | 8px | Compact control padding, row metadata gaps |
| md | 16px | Default element spacing, panel padding, row horizontal padding |
| lg | 24px | Route padding, sidebar inner padding, preview section spacing |
| xl | 32px | List/preview column gap, primary content gutters |
| 2xl | 48px | Major empty/error state offset inside the route |
| 3xl | 64px | Reserved for later page-level spacing; avoid in compact Phase 2 panels |

Exceptions: 44px minimum interactive hit area for nav items, session rows, and header controls. This is a hit-area rule, not a spacing token.

## Typography

Use exactly four sizes and two weights in Phase 2. Do not scale font size with viewport width. Letter spacing is `0`.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 14px | 400 | 1.4 |
| Body | 16px | 400 | 1.5 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Usage rules:

| Role | Usage |
|------|-------|
| Label | Nav entries, column labels, badges, timestamps, secondary metadata |
| Body | Session titles, preview text, empty/error bodies |
| Heading | Route title, preview panel title, empty/error headings |
| Display | App-level shell title only if present; do not use inside rows or compact panels |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8FAFC` | App background and route canvas |
| Secondary (30%) | `#FFFFFF` | Sidebar, header, session list surface, preview surface |
| Accent (10%) | `#2563EB` | Active Sessions nav item, selected session row indicator, focus rings, primary CTA |
| Destructive | `#DC2626` | Error states and future destructive confirmations only |

Accent reserved for: active Sessions nav item, selected session row indicator, keyboard focus rings, and the primary `Reload Sessions` CTA. Do not use accent for generic counts, disabled nav placeholders, status badges, or decorative gradients.

Supporting neutrals:

| Token | Value | Usage |
|-------|-------|-------|
| Text strong | `#111827` | Primary readable text |
| Text muted | `#64748B` | Metadata, disabled placeholder nav, secondary labels |
| Border | `#CBD5E1` | Panel dividers, row separators, badge outlines |
| Unknown/unsupported fill | `#F1F5F9` | Explicit `Unknown` and `Unsupported` badges |

State color rule: failed/error states may use destructive red. Unsupported and unknown states must use neutral outlined badges with explicit text, not green success styling.

## Layout Contract

| Area | Contract |
|------|----------|
| Window | Minimum useful viewport is 1024px by 680px. Content must remain usable down to 800px width by collapsing the preview below the list. |
| Shell chrome | Use a persistent left sidebar at 224px on desktop. Header height is 56px. Sidebar and header use secondary surface with a single border divider. |
| Navigation | Show Overview, Projects, Sessions, and Diagnostics. Sessions is active and navigable. Other entries are disabled placeholders with muted styling and `aria-disabled="true"`. |
| Route | Default route is `/sessions`. The page title is `Sessions`. |
| Sessions layout | Desktop uses two columns: session list minmax 360px to 42%, preview minmax 0 to 1fr. At widths below 900px, stack list above preview. |
| Session list | Render sessions as dense rows separated by borders, not decorative cards. Row height is at least 64px. Selected row has an accent left border and subtle `#EFF6FF` fill. |
| Preview panel | Render the selected session summary, harness badge, lifecycle/status label, timestamps, capability warnings, and a short evidence summary. Do not render full timeline or Run Audit panels in Phase 2. |
| Empty/error areas | Center within the route content area using `2xl` vertical offset. Do not use marketing hero layout, oversized illustration, or explanatory feature prose. |

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Initial load | Show a compact skeleton with three session rows and one preview block while `sessions.list` is pending. |
| Session selection | Clicking a row selects it and updates the preview in place. Arrow keys move through rows; Enter selects the focused row. |
| Primary CTA | `Reload Sessions` re-runs the read-only session list request through preload/IPC. It must not scan arbitrary roots or mutate data. |
| Disabled nav | Placeholder nav entries do not navigate. Use tooltip copy `Available in a later phase`. |
| Unsupported states | Show explicit badges: `Unsupported` for unavailable capabilities and `Unknown` for missing or indeterminate evidence. Never replace missing evidence with `0`. |
| Harness display | Show adapter display names from view-model metadata only. Do not branch on provider IDs in renderer code. |
| Error handling | Show sanitized user-facing errors only. Do not expose raw stack traces, raw filesystem records, adapter-private object names, or shell command details. |
| Read-only scope | No UI controls for launch, approve, reject, terminal input, PR creation, cleanup, delete, reset, or source mutation are allowed in Phase 2. |
