<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| normalized shell evidence -> shared shell parser | Shared core must not smuggle adapter-specific assumptions into shell parsing just because Gemini is the first real adapter. |
| scan-time artifact loading -> persisted derived summaries | Output artifacts may be missing, malformed, or unreadable; derivation must preserve degraded evidence without caching full bodies or inventing success. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering | shell truth derivation | mitigate | Make explicit shell `exitCode` authoritative, parse artifacts during scan while safe bindings are live, and capture parser diagnostics instead of trusting raw tool status. |
| T-05-01-02 | Spoofing | shared evidence links | mitigate | Add harness-neutral relation fields (`toolCallId`, `artifactIds`, `rawToolStatus`) rather than Gemini-only event-order heuristics. |
| T-05-01-03 | Information Disclosure | cache persistence | mitigate | Persist parsed summaries and derived facts only; leave full stdout/stderr bodies lazy in output artifacts. |
| T-05-01-SC | Tampering | npm/pip/cargo installs | mitigate | No package install is planned. If implementation attempts any install, stop and add a blocking human verification checkpoint because slopcheck was unavailable. |
</threat_model>
