<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer -> preload -> IPC | User-entered adapter/path/source actions cross from untrusted renderer into main. |
| IPC -> source/scanner/cache services | Validated commands can change local app registry/cache state but must remain read-only toward harness/repo roots. |
| main service -> renderer DTOs | Diagnostics and paths must be sanitized and capability truth must not be flattened. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07-01 | Elevation of Privilege | `preload/index.ts` | mitigate | Expose only named source methods; no generic `ipcRenderer`, filesystem, shell, Electron, or invoke bridge. |
| T-03-07-02 | Tampering | `sources:*` IPC handlers | mitigate | Validate all requests with strict Zod schemas and restrict operations to local registry/cache state; no source-root mutation. |
| T-03-07-03 | Information Disclosure | `data-sources-view-model-service.ts` | mitigate | Return sanitized diagnostics and DTOs only; no raw stack traces, raw filesystem dumps, raw events, or adapter-private object names. |
| T-03-07-04 | Repudiation | scan/validation state | mitigate | Persist validation, scan, cache, diagnostics, and timestamps so failures and stale state remain visible. |
| T-03-07-SC | Tampering | npm/pip/cargo installs | mitigate | No package install is planned in this plan. If implementation attempts any install, add a blocking human verification checkpoint because slopcheck was unavailable. |
</threat_model>
