| Boundary | Description |
|----------|-------------|
| renderer/preload -> main source registry | Future user-entered paths become untrusted IPC payloads and registry records. |
| registry store -> app data filesystem | Local JSON persistence can be malformed, stale, or tampered with. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | `source-registry-store.ts` | mitigate | Validate persisted `sources.json` with Zod, reject malformed records into diagnostics, and preserve unknown state rather than coercing to success. |
| T-03-01-02 | Spoofing | source IDs | mitigate | Generate source IDs from adapter identity plus native/root identity using existing stable ID helpers per DATA-05. |
| T-03-01-03 | Repudiation | source validation state | mitigate | Persist validation result summary, diagnostics, and timestamps so failures remain visible after reload per D-05. |
| T-03-01-SC | Tampering | npm/pip/cargo installs | mitigate | No package install is planned. If implementation attempts any install, stop and add a blocking human verification checkpoint because slopcheck was unavailable. |
</threat_model>
