<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer/preload -> main source registry | Future user-entered paths become untrusted IPC payloads and registry records. |
| shared scanner -> adapter | Adapter receives scoped helpers and source records but must not gain arbitrary filesystem access. |
| registry store -> app data filesystem | Local JSON persistence can be malformed, stale, or tampered with. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | `source-registry-store.ts` | mitigate | Validate persisted JSON with Zod, reject malformed records into diagnostics, and preserve unknown state rather than coercing to success. |
| T-03-01-02 | Information Disclosure | `safe-filesystem.ts` | mitigate | Canonicalize real paths and enforce configured source-root allowlists before stat/read/list operations. |
| T-03-01-03 | Elevation of Privilege | `AdapterContext` | mitigate | Expose narrow read-only helpers only; adapters do not receive raw `fs`, shell execution, cache write, or watcher lifecycle authority. |
| T-03-01-04 | Spoofing | source IDs | mitigate | Generate source IDs from adapter identity plus native/root identity using existing stable ID helpers per DATA-05. |
| T-03-01-SC | Tampering | npm/pip/cargo installs | mitigate | No package install is planned. If implementation attempts any install, stop and add a blocking human verification checkpoint because slopcheck was unavailable. |
</threat_model>
