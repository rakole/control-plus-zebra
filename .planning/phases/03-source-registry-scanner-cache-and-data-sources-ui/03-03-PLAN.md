<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scanner -> adapter | Shared scanner invokes untrusted adapter output and raw event streams. |
| adapter output -> normalized store | Malformed or spoofed normalized entities could poison session state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | `normalization-validator.ts` | mitigate | Validate adapter/source identity and relationships before merge/cache writes. |
| T-03-03-02 | Spoofing | `session-merger.ts` | mitigate | Keep global session identity adapter/source-aware and test cross-source native ID collisions. |
| T-03-03-03 | Information Disclosure | `scanner.ts` | mitigate | Scanner builds scoped adapter contexts from safe filesystem helpers; adapters receive no broad filesystem access. |
| T-03-03-04 | Denial of Service | `raw-artifact-index.ts` | mitigate | Track changed artifacts and skip unchanged parse paths where possible; preserve stale/unknown states if metadata is indeterminate. |
| T-03-03-SC | Tampering | npm/pip/cargo installs | mitigate | No package install is planned. If implementation attempts any install, stop and add a blocking human verification checkpoint because slopcheck was unavailable. |
</threat_model>
