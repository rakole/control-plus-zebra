<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| normalized diagnostics -> renderer DTOs | Diagnostics can contain sensitive implementation detail and must be sanitized before display. |
| shared status vocabulary -> all triage routes | Inconsistent labels across routes can mislead users into thinking missing evidence is safe on one page and broken on another. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04-01 | Information Disclosure | diagnostics DTOs | mitigate | Return concise sanitized code, severity, source area, and message fields only; exclude raw stack traces, raw shell output, and cache payloads. |
| T-06-04-02 | Spoofing | truth-state badges | mitigate | Centralize shared truth-state and capability-warning presentation so unsupported or unknown states cannot silently diverge across routes. |
| T-06-04-03 | Tampering | provider-specific diagnostics behavior | mitigate | Drive filters and labels from metadata and shared enums only; do not branch on provider IDs in shared renderer or main code. |
| T-06-04-SC | Tampering | UI package install | mitigate | Use existing CSS and approved primitives only; no new dependency installs are required for Diagnostics or truth-state components. |
</threat_model>
