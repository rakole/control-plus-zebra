| renderer navigation -> preload methods | New routes increase bridge surface area and must not introduce generic IPC escape hatches or raw cache leaks. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Spoofing | triage summaries | mitigate | Build overview, project, and session truth only from main-owned DTOs backed by normalized plus derived cache data; never infer git or audit truth from labels or text snippets. |
| T-06-01-02 | Information Disclosure | IPC and preload growth | mitigate | Add one strict request/response schema and one preload method per surface; no raw cache records, generic invoke helpers, or unsanitized path leakage. |
| T-06-01-03 | Tampering | truth-state rendering | mitigate | Model unsupported and unknown values explicitly in DTOs and assert in renderer tests that missing evidence never renders as `0`, `Passed`, or `Clean`. |
| T-06-01-SC | Tampering | npm or UI package install | mitigate | Reuse existing CSS and already-configured official primitives only; if implementation needs a new dependency, stop and require a human checkpoint first. |
